# 投标配图工具重构计划

> 目标:把配图工具从「自由生成 SVG」重构为「**冻结 SVG 模板 + 契约先行生成 + 决策核心 + 三级优雅降级**」的体系,在真实投标内容上**稳定**产出**精美**、规范的配图。

---

## 1. 背景与问题

当前 illustration 已是 v2 平台化结构,但配图主要依赖 SVG 自由生成——坐标、图标、箭头、配色全部由模型临场决定,**输出效果无法保障**(文字爆框、元素重叠、连线穿框、版式不专业)。

根因不是 SVG 本身,而是「自由生成」:把整张图的布局命运交给了模型。一旦改为由设计师预先画好、运行时几何零改动的 **SVG 模板**,SVG「无自动布局、逐元素手工控制」的特性就从缺点变为优点——图标、箭头、背景、文字框全部可被友好控制。

---

## 2. 重构目标与设计不变式

**北极星:** 大多数图走模板路径,精美且稳定;任何图都出得来,最差有兜底,绝不开天窗。

**六条设计不变式(贯穿全程的纪律):**

1. 模板 = 冻结的 SVG 几何 + 自描述契约;运行时几何零改动。
2. LLM 只做两件语义活——**选路径/选模板**、**按契约生成内容**;其余全部确定性软件。
3. 合规在生成时就保证(适配前移),渲染器保持哑;文字爆框在到达渲染器之前被拦截。
4. 外部契约稳定:`job` 进、`manifest` 出;调用方不依赖内部模板 / 渲染器 / 文件命名。
5. SVG 模板为主力;echarts 管数据图;graphviz / mermaid 兜大型自动布线;自由矢量 SVG 仅作最后兜底。
6. 主题与几何正交:换色走变量,不动布局。

---

## 3. 非目标(本轮明确不做)

- 参考图自动生成模板——后置到模板规范稳定之后,且应为「参考图 → 草稿模板 → 人工精修」的人机协作,不前置。
- 28 类图型一次性全模板化——按高频逐类推进。
- 替换现有 SVG 渲染能力——复用 `svg_renderer` / `svg_v2`,而非重写。

---

## 4. 核心架构

### 4.1 决策层是核心

本工具真正的难点和价值不在渲染,而在**有效决策**:基于 LLM 与「工具能力目录」,为每张图选出最合适的产出路径,并在最优路径不可行时优雅降级。决策层的输出不是「哪个渲染器」,而是「**走哪一级产出路径**」。

### 4.2 模板 = 冻结 SVG 几何 + 自描述契约

一个 SVG 模板分两层:

- **设计师冻结层(几何 + 视觉):** 背景与装饰底图、图标库(位置/尺寸固定)、箭头与连线样式、文字框形状。一次设计,永久复用。
- **软件运行时填充层:** 文字内容注入、节点按数据增减、主题色变量切换、受约束改写适配。

每个模板**自带一份契约**,供决策与生成使用:

- **范式描述(给 LLM 理解语义):** 自然语言说明这张图表达什么、怎么表达(如「三层分层架构,自上而下表示调用层级」)。
- **内容结构 schema(给 LLM 约束、给软件校验):** 形式化槽位 + 硬约束(层数、节点数、字数上限、图标白名单等)。

### 4.3 契约先行的两段式生成

把适配从「事后修」前移到「生成时即合规」:

1. **选模板(LLM,语义判断):** 输入原始内容 + 能力目录摘要,LLM 选出最合适的模板。选模板时手上只有散文,这种「内容 → 视觉范式」的映射只有 LLM 能做。
2. **按契约生成(LLM,受约束):** 系统回给 LLM 选中模板的范式 + 结构 schema,LLM 产出**严格满足该 schema** 的填充内容,而非自由发挥。
3. **确定性校验门禁:** 逐条核对必填槽位、字数、节点数、图标白名单。不过则把**具体违规项**反馈 LLM 重试;仍不行则降级。
4. **注入哑渲染器:** 已保证合规的数据注入冻结的 SVG 几何,出图。

`fit_score` 作为对 LLM 选择的**确定性复核**(如 LLM 选 3 节点模板但内容明显 8 节点,则驳回或提示换变体)。

### 4.4 三级优雅降级(决策核心的关键)

决策层永远从最可控的一级开始往下试,只在上一级确实不可行时降级,每一步降级都留痕:

| 级别 | 路径 | 设计可控性 | 触发条件 | 保证 |
|---|---|---|---|---|
| **Tier 1** | SVG 模板 + 契约生成 | 最强(首选) | 内容能落进某模板契约 | 精美 + 稳定 |
| **Tier 2** | 柔性渲染器(echarts / mermaid / graphviz / html_css) | 中 | 无模板契约可装下 | 靠渲染器自带安全网兜底 |
| **Tier 3** | 自由矢量 SVG | 最低(最后手段) | 前两级均不可用 | 一定出图,标记需人工复核 |

- **Tier 2 的鲁棒性来自渲染器自带的安全网:** echarts 数据驱动自适应;mermaid 文法合法即出图;graphviz 自动布线;html_css 浏览器自动版式。决策层按内容信号 + LLM 判断选其一。
- **Tier 3 仅保证不开天窗:** 质量靠不住,manifest 必须标记「兜底产出,建议人工复核」。

### 4.5 工具能力目录(决策层的输入)

决策层消费一份新产物——**能力目录**,登记:

- 所有 SVG 模板:用途摘要 + 容量摘要 + 完整契约。
- 各柔性渲染器的能力描述:擅长/不擅长什么、输入要求。

LLM 选路径、软件做降级判断,均依赖这份目录。它与模板注册表共址,但范围更大(模板 + 渲染器)。

### 4.6 与现有模块的映射

| 职责 | 落点 | 改动性质 |
|---|---|---|
| 核心决策层(选路径 / 选模板 / 降级) | `core/decision.py` + `core/router.py` | 升格为一等核心 |
| 槽位绑定 + 受约束改写适配 | `core/normalize.py` | 扩展 |
| 模板与能力目录 | `core/registry.py` 扩展 | 新增 |
| 数据级校验门禁 | `core/validator.py` | 扩展 |
| SVG 模板注入渲染 | `renderers/svg_v2.py`(复用) | 复用 |
| 数据图 | `renderers/echarts_v2.py` | 既有 |
| 装配契约留痕 | `core/manifest.py` | 增字段 |
| 两段式生成编排 | `pipeline/stages/s5_illustrate.py` | 升级 |

外部入口(`api`、CLI、pipeline)共享同一契约:`job` 进、`manifest` 出。

## 5. 关键数据结构(草案)

```text
Template
  id            : str            # 如 arch.layered.v2
  diagram_type  : str
  variant_of    : str            # 同图型变体分组
  svg_geometry  : ref            # 冻结的 SVG(背景/图标/箭头/文字框/槽位标记)
  contract      : Contract
  theme_vars    : ThemeSchema    # SVG 内引用的 CSS 变量

Contract
  paradigm      : str            # 范式描述(自然语言,给 LLM)
  structure     : Schema         # 形式化槽位
  constraints   : {字数上限 / 节点数区间 / 图标白名单 / 必填槽位 ...}

CapabilityCatalog
  templates     : [TemplateSummary]   # 摘要 + 容量,供选模板
  renderers     : [RendererCapability] # 柔性渲染器能力描述,供降级

PlanResult (api.plan 返回)
  id, type
  decision: { tier, template, renderer, theme, fit_score,
              degraded_from?, fallback? }

ManifestEntry (新增字段)
  template, theme, fit_score, tier, degraded_from, needs_human_review
```

## 6. 端到端流程

```
job 内容
  → 校验
  → 决策层(LLM + 能力目录)
       Tier 1: 选 SVG 模板 → 取契约 → 按契约生成 → 数据级校验
                校验不过 → 反馈违规项重试
                变体阶梯:换更大变体 → 拆成两图 → 通用保底模板
       整层不可行 → Tier 2: 按内容信号 + LLM 选柔性渲染器
       仍不可行 → Tier 3: 自由矢量 SVG(标记需人工复核)
  → 注入对应渲染器出图
  → 渲染后 QA 兜底(像素测量:溢出/重叠/对比度)
  → 写 manifest(全程降级留痕)
```

LLM 只出现在「选模板/选路径」与「按契约生成」两处,且输出均可被确定性校验。

## 7. 分阶段实施计划

### Phase 0 · 地基与止血

**目标:** 主链路干净、抽象与决策核心就位、能跑测试。

**任务:**
- 快照当前未提交改动,拉基线分支,区分「已有改动」与「重构新增」。
- 修断点 1:`api.plan()` 返回含最终 `renderer` 与 `decision`(含 `tier` / `degraded_from`)。
- 修断点 2:`api.render()` 增 `export_echarts`,透传到 `platform.render_job / render_job_file`。
- 处理 `_fix_fonts.py`:确认是否在主链路,不在则移出编译门禁或修语法。
- 搭测试环境(补 `pytest`),跑通现有 illustration 测试作回归基线。
- 定义抽象(只接口、不含具体模板):`Template`、`Contract`、`CapabilityCatalog`、`TemplateRegistry` 扩展;`manifest` 增字段。
- 将 `decision.py` + `router.py` 明确定位为「核心决策层」,定义能力目录数据结构。

**涉及:** `api.py`、`platform.py`、`core/registry.py`、`core/decision.py`、`core/router.py`、`core/manifest.py`、`_fix_fonts.py`。
**验收:** `compileall` 全绿;现有测试通过;新签名有单测;空目录/抽象可 import。

### Phase 1 · 单图型纵向贯通(系统架构图)

**目标:** 一个图型端到端跑通 Tier 1 主路,在真实内容上产出一张稳定精美的架构图。**窄而深,验证可行性。**

**任务:**
- 手工精修 1 个 SVG 模板变体(`arch.layered.v2`,3 层),冻结背景/图标/箭头/文字框,槽位用占位标记。
- 写它的契约:范式描述 + 结构 schema(层数、每层节点数、字数上限、图标白名单)。
- 注册表登记:svg + contract + 目录摘要。
- `s5_illustrate` 升级为两段:选模板(LLM 读目录摘要)+ 按契约生成(LLM 读完整 schema)。
- 确定性校验门禁:必填/字数/节点数/图标白名单;不过则反馈违规项重试。
- SVG 填充渲染:复用 `svg_v2`,把校验过的数据注入冻结模板。
- 主题:SVG 用 CSS 变量挂一套默认色。
- `s6` 读 manifest 插图,验证打通。

**涉及:** `s5_illustrate.py`、`core/router.py`、`normalize.py`、`renderers/svg_v2.py`、schemas、registry、manifest。
**验收:** 多份真实架构描述生成,零爆框/重叠/穿框;校验失败能自动重试到合规;manifest 字段完整;S6 章节正确插图。

### Phase 2 · 规范固化 + 三级降级 + 稳定性硬化

**目标:** 形式化「合格模板/契约」;建成完整三级降级链;稳定性可回归。**本轮稳定性的核心阶段。**

**任务:**
- 写《模板与契约规范》:契约必含项、槽位定义法、约束表达、主题变量结构、命名与版本规则。
- 架构图补容量变体(3/6 节点、带分组);`fit_score` 复核 LLM 选择。
- 建成完整降级链:Tier 1 变体阶梯(换更大变体 → 拆分 → 保底模板)→ Tier 2 柔性渲染器(内容信号 + LLM 选择)→ Tier 3 自由 SVG;每步降级写进 manifest,Tier 3 强制标记需人工复核。
- 渲染后像素 QA(溢出/重叠/对比度)作为最后兜底,接入 `s7_verify`。
- 每模板黄金回归快照,纳入 CI。

**涉及:** `core/router.py`、`core/decision.py`、`core/validator.py`、`s7_verify.py`、测试。
**验收:** 规范评审通过;超容量内容能正确逐级降级并留痕;切 3 套主题版面不变;回归全绿。

### Phase 3 · 横向扩类(绞杀者式)

**目标:** 把稳定的模板+契约模式复制到下一批高频图型;路由「有模板走 Tier 1,否则自动降级」,不阻塞流水线。

**任务:**
- 按规范为 流程图、组织结构、技术路线 各做 1–2 变体 + 契约。
- `router`:命中 SVG 模板走 Tier 1,未命中走 Tier 2/3;新图型即使无模板也能出图。
- echarts 仅数据图;graphviz/mermaid 仅大型自动布线。
- 每类纳入回归。

**涉及:** registry、router、`renderers/echarts_v2.py`、各 stage。
**验收:** 每个新图型达 Phase 1 同等质量基线;老图型零回归;无模板图型靠降级正常出图。

### Phase 4 · 工具化与降本(后置 / 可选)

- 模板制作工具:自动校验契约与 SVG 槽位一致性。
- 参考图 → 草稿模板辅助:人机协作产出 `Template` 实例再人工精修。**仅在模板规范稳定后启动。**

## 8. 完整降级链(汇总)

> **变体阶梯 → 拆分 → 保底模板(以上属 Tier 1)→ 柔性渲染器(Tier 2)→ 自由矢量 SVG(Tier 3)**

决策层基于 LLM + 能力目录从上往下逐级尝试:**首选精美,保底不失败,全程留痕。**

---

## 9. 量化验收基线

- 选定图型在真实投标样本上,数据级校验 + 渲染 QA 通过率达标。
- 零文字爆框 / 零元素重叠 / 零穿框。
- 同一张图切换 3 套主题,版面不变。
- 每模板回归用例全绿。
- 任意内容均能出图(最差落 Tier 3 并标记人工复核),无开天窗。

---

## 10. 风险与排期建议

- **关键路径是 Phase 1。** 一旦跑通,后面是复制。
- **最大实现风险集中在两段式 LLM 交互的提示词工程**,以及契约 schema 的表达力。Phase 1 只追求「一个变体、一条主路打通」,变体阶梯与降级留到 Phase 2,不在验证闭环前铺开。
- 每个阶段结束都在**真实投标样本**上验收,不只看示例。
- 扩类不阻塞:Phase 3 起,新图型无模板也能靠降级出图,逐类补模板可从容推进。

## 12. 术语

- **契约(Contract):** 模板自带的「范式描述 + 结构 schema + 约束」,既驱动 LLM 生成,又供软件校验。
- **能力目录(Capability Catalog):** 决策层的输入,登记所有模板与渲染器能力。
- **Tier 1/2/3:** 三级降级路径,可控性递减、通用兜底性递增。
- **哑渲染器:** 只接收已校验合规的数据并注入冻结几何,不做内容决策。
