# ECharts 本地图表工作台测试报告

> 测试日期：2026-05-27  
> 实现方式：本地 HTML + Apache ECharts 5.6.0  
> 页面入口：[index.html](<../web/echarts-workbench/index.html>)  
> 测试产物：[ECharts_HTML_workbench_20260527](<../results/tests/ECharts_HTML_workbench_20260527/>)

## 测试目标

验证在不引入后台服务的条件下，AI 输出受约束的数据图 JSON，本地 HTML 页面即可完成标书用复杂数据图的预览与 SVG/PNG 导出。

## 已实现能力

| 项目 | 结果 |
| --- | --- |
| 离线运行 | 页面引用本地 `vendor/echarts.min.js`，无 CDN 依赖 |
| AI 输入协议 | 新增 `schema/echarts_diagram.schema.json` |
| 示例描述 | 新增 `examples/示例_ECharts数据图描述.json` |
| 图型覆盖 | 柱线组合图、雷达图、Sankey 流向图 |
| 页面功能 | 图件切换、JSON 导入、恢复演示数据、当前 JSON 下载 |
| 导出能力 | SVG 和 2 倍分辨率 PNG |

## 实际验证结果

| 图型 | 页面渲染 | SVG 导出 | PNG 导出 | 判断 |
| --- | ---: | ---: | ---: | --- |
| 柱线组合图 | 通过 | 通过 | 通过 | 适合呈现规模与达标率趋势 |
| 雷达图 | 通过 | 通过 | 通过 | 适合呈现多维能力响应 |
| Sankey 流向图 | 通过 | 通过 | 通过 | 适合呈现数据汇聚与应用流向 |

补充验证：

- JSON Schema 已对三张示例图校验通过。
- 使用浏览器导入示例 JSON 后，页面提示“已导入 3 张 AI 数据图描述”。
- 浏览器页面会拒绝数量不一致或引用无效的 AI 数据图描述，避免错误图件进入导出环节。
- 浏览器运行过程未产生脚本错误。
- 演示数据仅用于验证排版和图表能力，不代表实际项目统计口径。

## 结论

ECharts 以本地 HTML 页面承载已经满足当前标书场景中“AI 生成描述、人工快速审图、导出正式图片”的数据图需求。它应作为原生 SVG 框图工具的补充：架构、流程和时序类框图继续使用 SVG 引擎，数值趋势、能力评价和数据流量关系优先使用本工作台。
