# Project Docs

本目录存放项目级文档。源码目录 `src/` 只承载运行时代码、Schema、Prompt、模板契约和必要的包内资源；长期背景、架构、计划、标准和交接信息统一放在 `docs/`。

## 目录

| 路径 | 作用 |
| --- | --- |
| `docs/NAVIGATION.md` | 人和智能体按任务类型快速定位文件 |
| `docs/FILE_GOVERNANCE.md` | 文件归位、命名、Git 跟踪和产物治理 |
| `docs/PROJECT_BUILD_OVERVIEW.md` | 工程项目级建设总控：目标、现状、阶段计划、风险、下一步 |
| `docs/PROJECT_FEATURES.md` | 已完成功能、成熟度、入口、限制和验证方式 |
| `docs/PROJECT_TODO.md` | 全工程 TODO 队列、优先级、状态和验收 |
| `docs/PROJECT_STRUCTURE.md` | 项目文件结构与放置规则 |
| `docs/workspace/` | 项目记忆系统：状态、路线图、决策、交接 |
| `docs/architecture/` | 架构设计、重构方案、系统工程计划 |
| `docs/standards/` | 内容标准、绘图标准、质量标准 |
| `docs/quality-targets/` | 人工参考目标、质量评分、持续 Todo 和评审记录 |
| `docs/methodology/` | 从本项目实践提炼的方法论 |
