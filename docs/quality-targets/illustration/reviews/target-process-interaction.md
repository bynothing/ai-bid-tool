# Target Review: Process Interaction Map

日期：2026-06-07

## 基本信息

- 原型：A 工艺流程 / 系统交互大图
- 参考图：`references/五菱总装A-H工艺段流程及系统交互点分析图 (1).png`
- 目标用途：复杂流程、系统交互点、工艺段关系、主辅流程、术语说明

## 目标图值得学习的地方

### 信息表达

- 图题明确，副标题解释分析范围。
- A-H 八个工艺段分区明确。
- 每个分区内部只展开本段关键步骤。
- 系统交互点用颜色和标签标注。
- 右侧图例说明交互点类型、流程箭头类型和配送方式。

### 版式结构

- 主体为分区卡片矩阵。
- 主流程用粗箭头在分区之间流动。
- 辅助流程和物料支持流程用虚线弱化。
- 说明区独立，不干扰主体流程。

### 箭头关系

- 主流程、物料支持、双向流程、汇入流程有不同线型。
- 箭头服务路径表达，不是所有节点之间都连线。
- 虚线关系避开主体文本，降低视觉噪音。

### 视觉层级

- 每个工艺段有独立色彩。
- 卡片标题条强，内部信息弱。
- 图例区用深色标题和浅色内容组织。
- 整体像工程说明图而不是普通流程图。

## 不学习的地方

- 不学习重金属装饰边框。
- 不学习过密小字。
- 不强制使用复杂背景。
- 不强制使用同样的颜色数量。

## 当前系统预计差距

| 优先级 | 差距 | 工程层级 | 建议任务 |
| --- | --- | --- | --- |
| P0 | 缺少“工艺段/业务段分区 + 段内步骤”的专用契约 | Job 契约 | 设计 `sections[] + steps[] + interaction_types[]` |
| P0 | 主流程和辅助流程没有分层 | 连线层 | 增加 `primary_flow` / `support_flows` |
| P0 | 箭头容易穿插和交叉 | 布局/连线 | 固定主路径通道和辅助线型 |
| P1 | 缺少图例和术语说明区 | 布局层 | 增加 legend/glossary 区域 |
| P1 | 复杂信息缺少容量治理 | 决策层 | 超过阈值拆图或摘要 |
| P2 | 视觉不够工程化 | 视觉层 | 建立标题条、卡片、语义色 tokens |
