# Illustration Acceptance Rubric

本文件定义“工程级标书配图”的评分标准。后续优化不只看是否生成文件，而是按下列维度打分。

总分 100 分。80 分以上才可视为可交付，90 分以上才可视为目标级。

## 1. 信息清晰度（25 分）

| 分值 | 标准 |
| --- | --- |
| 0-10 | 主题不清楚，节点堆砌，看不出图要表达什么 |
| 11-18 | 能看出大概主题，但主线、层级或边界不清楚 |
| 19-22 | 主题明确，主线基本清楚，局部信息仍需解释 |
| 23-25 | 5-10 秒内能理解图的主题和核心结构 |

检查点：

- 图题与主体内容一致。
- 主路径或核心结构突出。
- 每个区域都有明确职责。
- 没有无意义节点和装饰性信息。

## 2. 结构与布局（20 分）

| 分值 | 标准 |
| --- | --- |
| 0-8 | 元素散乱，层级混乱，对齐和间距明显失控 |
| 9-14 | 基本成图，但局部拥挤或重心不稳 |
| 15-18 | 分区、层级、对齐和间距稳定 |
| 19-20 | 结构像专业工程图，扫描路径自然 |

检查点：

- 容器、层级、区域边界清楚。
- 节点间距稳定。
- 视觉重心平衡。
- 不把卡片塞满整张画布。

## 3. 箭头与关系表达（20 分）

| 分值 | 标准 |
| --- | --- |
| 0-8 | 箭头混乱、交叉多、方向不清、语义不明 |
| 9-14 | 主要方向可辨认，但存在穿线、重复线或标签混乱 |
| 15-18 | 主次关系清楚，连线有秩序，标签克制 |
| 19-20 | 箭头少而准，关系表达专业、清爽、可追踪 |

检查点：

- 主路径优先。
- 避免中心区域多线交叉。
- 箭头不穿越节点文字。
- 标签短而有信息量。
- 同类关系线型一致。

## 4. 文本与术语（15 分）

| 分值 | 标准 |
| --- | --- |
| 0-5 | 文本过长、截断、爆框或术语空泛 |
| 6-10 | 文本基本可读，但长短不均、信息密度不稳 |
| 11-13 | 节点标题短而具体，说明文字克制 |
| 14-15 | 文本像工程标书语言，准确、短、可读 |

检查点：

- 节点标题优先使用名词短语。
- 少用空泛词。
- 不在节点里写长句。
- 重要协议、数据类型、状态标签可保留。

## 5. 视觉专业度（15 分）

| 分值 | 标准 |
| --- | --- |
| 0-5 | 像调试输出或临时草图 |
| 6-10 | 有基本样式，但风格不统一或不够正式 |
| 11-13 | 正式、克制、适合标书 |
| 14-15 | 接近专业咨询/工程方案图观感 |

检查点：

- 色彩克制，支持层级。
- 字体、边框、圆角、图标风格统一。
- 有适当留白。
- 不使用花哨背景和无意义装饰。

## 6. 交付可用性（5 分）

| 分值 | 标准 |
| --- | --- |
| 0-2 | PNG/SVG 嵌入 Word 后不可读或尺寸失控 |
| 3-4 | 可嵌入，但局部文字偏小或清晰度不足 |
| 5 | Word 中清晰、可打印、可审阅 |

## 评级

| 总分 | 评级 | 处理 |
| --- | --- | --- |
| 90-100 | A | 目标级，可作为模板参考 |
| 80-89 | B | 可交付，继续微调 |
| 65-79 | C | 勉强可用，必须记录缺陷 |
| 0-64 | D | 不可交付，必须进入 Todo |

## 强制失败项

出现任一项，直接判为不可交付：

- 文字爆框或严重重叠。
- 箭头方向与业务语义相反。
- 关键节点缺失。
- 图题和内容不一致。
- 输出图片嵌入 Word 后无法阅读。
