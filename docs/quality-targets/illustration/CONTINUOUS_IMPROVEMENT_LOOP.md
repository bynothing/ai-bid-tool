# Continuous Improvement Loop

最后更新：2026-06-07

本文件定义配图质量从参考目标到工程实现的持续闭环。

## 闭环总览

```text
1. 目标输入
   人工参考图 + 目标说明

2. 目标抽象
   提炼图型原型、布局规则、箭头规则、视觉规则

3. 当前基线
   用当前系统生成同类图，保存输出和 manifest

4. 差距评分
   按 Rubric 对目标图和系统图分别评分

5. 任务拆解
   差距转为 Todo：数据契约、决策、布局、连线、视觉、QA

6. 开发实现
   小步修改模板、渲染器、能力目录、prompt 或 QA

7. 自动验证
   pytest + smoke + fixture + manifest 检查

8. 人工复评
   重新打分，记录是否接近目标

9. 沉淀回归
   好结果成为 fixture / template / QA rule

10. 下一轮
   从最高价值差距继续迭代
```

## 每轮迭代输入

每一轮必须明确：

- 本轮目标原型：A/B/C/D 或新增原型。
- 本轮参考图。
- 本轮当前系统输出。
- 本轮要提升的评分维度。
- 本轮不解决的问题。

推荐每轮只选 1 个原型、1-2 个质量维度。

## 每轮迭代输出

每一轮结束必须产生：

- `reviews/target-*.md`：目标与差距分析。
- `generated-baselines/...` 或 `tests/fixtures/...`：当前输出。
- `TODO.md` 状态更新。
- `RECORD.md` 迭代记录。
- 如涉及代码：测试结果。

## 任务拆解规则

质量差距必须落到以下至少一个工程层：

| 层级 | 可改内容 | 示例 |
| --- | --- | --- |
| 内容策略 | 图型选择、是否拆图、节点数量、边数量 | 复杂接口图拆成总览 + 细节 |
| Job 契约 | data schema、字段含义、容量限制 | 增加 `primary_flow` / `support_flows` |
| 决策层 | renderer/tier/template/fallback | 复杂交互图优先 Draw.io 或目标模板 |
| 布局层 | 分区、卡片、层级、标题区、说明区 | 工艺段卡片 + 图例栏 |
| 连线层 | 锚点、走廊、主次线型、标签 | 主流程粗箭头，辅助虚线 |
| 视觉层 | design tokens、颜色、字体、边框、留白 | 工程蓝 + 语义色 |
| QA 层 | fit、重叠、交叉、标签重复、可读性 | 箭头穿文字检测 |

不能只写：

```text
优化美观。
```

必须写成：

```text
为 process.interaction_map 增加 primary_flow/support_flow 结构，
渲染时主路径使用粗箭头、辅助关系使用虚线，并在 manifest 记录边数量。
```

## 质量门禁

每轮至少检查：

- `python -m pytest`
- 对应 smoke job 可生成图。
- manifest 有 renderer/tier/fallback/quality warnings。
- 人工评分至少一个维度提升。

出现以下情况，本轮不能关闭：

- 箭头方向错误。
- 主路径无法辨认。
- 文字爆框或关键重叠。
- 只是视觉变漂亮，但信息更不清楚。

## 评审节奏

建议节奏：

- 每轮 1 个图型原型。
- 每轮 3-7 个 Todo。
- 每轮结束生成一个对比记录。
- 每 3 轮整理一次通用规则，沉淀到 `docs/standards/illustration/` 或代码 QA。

## 当前第一轮建议

第一轮优先选择：

```text
A. 工艺流程 / 系统交互大图
```

原因：

- 它最能暴露当前系统的核心短板：信息组织、箭头治理、图例和复杂流程表达。
- 它能同时推动 Job 契约、决策层、布局层、连线层和 QA。
- 改好这一类图，会显著提升标书中复杂方案表达能力。

第一轮建议只解决：

- 主流程与辅助流程分层。
- 分区卡片布局。
- 图例/说明区。
- 边标签去重。

暂不解决：

- 华丽背景。
- 拟物图标。
- 所有图型统一美化。
