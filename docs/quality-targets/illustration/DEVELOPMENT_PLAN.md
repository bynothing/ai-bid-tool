# Illustration Quality Development Plan

最后更新：2026-06-07

本计划用于把“配图不好看、不清楚、箭头混乱”转化为可持续开发任务。

## 总体策略

先建立目标，再做开发：

```text
人工参考目标
  -> 质量评分标准
  -> 当前系统基线
  -> 差距分析
  -> 分图型改造
  -> 自动 QA
  -> 回归 fixture
```

具体执行采用闭环：

```text
目标输入
  -> 目标抽象
  -> 当前基线
  -> 差距评分
  -> 任务拆解
  -> 开发实现
  -> 自动验证
  -> 人工复评
  -> 沉淀回归
  -> 下一轮
```

详见 `CONTINUOUS_IMPROVEMENT_LOOP.md`。

## Phase Q0：目标收集与基线建立

状态：进行中

目标：

- 收集人工参考图和目标说明。
- 记录当前系统输出的真实问题。
- 建立评分基准。

当前状态：

- `references/` 已有 87 张参考图。
- 已初步归纳 4 类目标原型，见 `REFERENCE_INVENTORY.md`。

任务：

- 从现有 87 张参考图中选择每类 1 张 A 级目标图。
- 每张 A 级目标图填写 `reviews/target-*.md`。
- 选 3-5 个当前系统输出作为基线，放入 `generated-baselines/` 或转为 fixture。
- 按 `ACCEPTANCE_RUBRIC.md` 打分。

验收：

- 至少 5 个参考目标有文字说明。
- 至少 3 个系统当前输出有评分和缺陷记录。
- `TODO.md` 中形成首批可执行任务。

## Phase Q1：表达结构先行

状态：待启动

目标：

先解决“信息表达不清楚”，再解决“美观”。

重点：

- 每类图明确固定表达模型。
- 限制单图表达的信息量。
- 建立节点、容器、主路径、说明区的组织规则。

开发方向：

- 在能力目录中增加图型表达策略字段。
- 为高频图型建立 data normalization 规则。
- 对节点和边做语义剪枝：少而准。
- 为图型选择增加“是否应拆图”的判断。

验收：

- 架构图、流程图、接口图各至少一个 fixture 达到信息清晰度 19 分以上。

第一轮优先：

- 工艺流程 / 系统交互大图。
- 输出目标：能表达多业务段、多系统交互点、主流程和辅助流程。
- 对应开发主题：`process.interaction_map` 或 `integration.process_interaction` 的 Job 契约和布局策略。

## Phase Q2：箭头和连线治理

状态：待启动

目标：

解决箭头混乱、交叉、穿越、标签重复的问题。

重点：

- 主路径优先。
- 次要关系弱化或进入说明区。
- 连线走廊和锚点固定。
- 标签短、少、必要。

开发方向：

- 为模板/Draw.io 渲染器建立 connector policy。
- 对边标签做规范化和去重。
- 对复杂关系图做自动分区或拆图。
- 对交叉过多的图强制降级到更合适渲染器或拆分。

验收：

- 箭头与关系表达评分达到 15 分以上。
- 不出现箭头穿越关键文本。

第一轮优先：

- 主流程粗箭头。
- 辅助流程虚线。
- 双向/循环流程单独线型。
- 图例区解释线型。
- 边标签去重和压缩。

## Phase Q3：视觉系统升级

状态：待启动

目标：

让输出观感达到正式标书图，而不是临时草图。

重点：

- 统一配色、字体、边框、阴影、图标、留白。
- 强化容器和层级。
- 建立一套工程蓝主视觉，但避免单调。

开发方向：

- 建立 design tokens。
- 重构模板组件视觉参数。
- 给 Draw.io 和 SVG 模板共享视觉规范。
- 建立高频图型的 A 级参考模板。

验收：

- 视觉专业度评分达到 11 分以上。
- 同一批图风格统一。

## Phase Q4：自动质量门禁

状态：待启动

目标：

把人工评审标准逐步转成自动 QA。

最低门禁：

- 文本 fit。
- 元素重叠。
- 箭头穿越关键文本。
- 边数量上限。
- 标签重复。
- PNG 可读性。

验收：

- QA 失败能写入 manifest。
- 测试中能复现至少 3 类质量失败。

## Phase Q5：持续回归

状态：待启动

目标：

每次优化都不破坏已有好图。

任务：

- 建立 `tests/fixtures/illustration_cases/`。
- 为每个图型保存输入、期望、评分、输出快照。
- 每次修复都补充 fixture。

验收：

- 高频图型都有可复现样本。
- 质量回退能被测试或人工评审记录发现。

## 第一轮执行计划

目标原型：

```text
A. 工艺流程 / 系统交互大图
```

输入：

- 参考图：`五菱总装A-H工艺段流程及系统交互点分析图 (1).png`
- 当前系统输出：待生成

本轮要解决：

1. 分区卡片布局。
2. 主流程和辅助流程分层。
3. 图例/术语说明区。
4. 边标签去重。
5. 超容量时拆图或摘要。

本轮不解决：

- 装饰性金属边框。
- 拟物设备图标。
- 所有图型统一重设计。

开发落点候选：

- `src/bid_tool/illustration_v2/core/models.py`
- `src/bid_tool/illustration_v2/core/decision.py`
- `src/bid_tool/illustration_v2/core/catalog.py`
- `src/bid_tool/illustration_v2/renderers/drawio.py`
- `src/bid_tool/illustration_v2/renderers/template_svg.py`
- `src/bid_tool/illustration_v2/templates/`
- `tests/fixtures/illustration_cases/`

验收：

- 生成一张同类系统交互图。
- Rubric 总分达到 70+，箭头与关系表达不低于 14。
- manifest 记录图型、renderer、边数量、降级和质量 warning。
- 至少一个 fixture 可复现。
