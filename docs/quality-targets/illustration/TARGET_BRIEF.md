# Illustration Target Brief

最后更新：2026-06-07

本文件由人工持续维护，用来描述自动配图最终要达到的目标。

## 目标一句话

自动配图应达到“工程级标书图”的效果：信息清楚、关系准确、箭头有秩序、版式稳定、视觉正式，评审人员无需读大段正文也能通过图理解方案结构和关键路径。

## 当前不满意点

- 图形看起来像临时拼接，不像正式工程交付图。
- 节点和容器层级不明显，主次关系弱。
- 箭头过多、穿插、方向混乱，不能表达清晰的数据流或业务流。
- 文本密度和卡片容量不稳定，有些节点信息过长。
- 图中元素虽然存在，但没有形成清楚的“故事线”。
- 同一批图之间风格不统一。

## 期望效果

### 信息表达

- 每张图只表达一个核心问题。
- 图题、主体结构、关键路径和结论能够相互呼应。
- 评审能在 5-10 秒内知道这张图讲什么。
- 评审能在 30 秒内看清主要模块、流程或接口关系。

### 版式结构

- 架构图应有清晰分层、分区、入口、能力层、数据层、外部系统和安全边界。
- 流程图应有明确起点、主线、分支、闭环和异常路径。
- 接口图应有系统边界、交互方向、协议/数据类型和安全控制。
- 拓扑图应有区域、链路、关键设备和边界关系。

### 箭头和连线

- 箭头必须服务信息表达，不应为了“显得复杂”而堆线。
- 主路径箭头优先，次要关系可用弱化线或说明区。
- 避免箭头穿越节点文字。
- 避免多条线在中心区域交叉。
- 同类关系使用一致线型和颜色。

### 视觉风格

- 正式、克制、工程化。
- 适合技术标书、政府/企业项目投标文件。
- 颜色支持层级和状态，不追求花哨。
- 字体、间距、边框、图标风格统一。
- 输出到 Word 后仍清晰、可读、可打印。

## 人工参考图要求

人工提供参考图时，不要求完全同领域，但必须说明：

- 这张图好在哪里。
- 希望学习它的哪些特征。
- 哪些特征不需要学习。
- 它对应哪类标书图型。

参考图放入：

```text
docs/quality-targets/illustration/references/
```

说明按：

```text
docs/quality-targets/illustration/REFERENCE_INTAKE_TEMPLATE.md
```

## 非目标

- 不追求营销海报风。
- 不追求装饰性复杂背景。
- 不把所有信息塞进一张图。
- 不让 LLM 直接自由生成复杂 SVG 坐标作为主路线。
- 不牺牲信息清晰度换取表面美观。
