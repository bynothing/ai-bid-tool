# 标书指定位置生成框图测试

## 测试目标

验证“由大模型直接生成标准化 SVG 框图，再输出 PNG 预览”的方案，是否适合作为标书编写过程中的日常插图能力。

## 模拟插入位置

| 项目 | 内容 |
| --- | --- |
| 标书章节 | `总体技术方案 / 系统总体架构设计` |
| 图题 | `甘肃省高风险放射源监管平台系统总体架构图` |
| 图形类型 | 分层系统总体架构图 |
| 输出风格 | 正式标书风格、蓝灰配色、横向分层、白底 |
| 数据依据 | 已确认的系统功能菜单模块 |

## 生成需求示例

```text
在《总体技术方案》的“系统总体架构设计”章节后插入一张系统总体架构图。

项目为“甘肃省高风险放射源监管平台”，请依据当前确认的功能模块生成正式标书风格的分层架构图：
- 服务对象：省级监管部门、涉源使用单位、运维人员、审批人员、系统管理员；
- 业务应用层：综合统计、放射源信息管理、实时状态监控、单位与源库管理、审批流程、运维管理、报表看板、国家数据对接；
- 平台支撑层：流程引擎、低代码配置、GIS 展示、统计报表、用户权限配置；
- 数据资源层：台账数据、单位人员车辆数据、定位轨迹数据、流程数据、交换日志数据；
- 基础保障层：应用服务、数据存储、身份认证、访问控制、日志审计与运维；
- 右侧表现国家平台、外省数据、定位终端、消息渠道等外部协同。

要求：输出 SVG 和 PNG，1600 像素宽，蓝灰商务风，图中不写未经确认的具体部署产品名称。
```

## 实测结果

| 核验项 | 结果 |
| --- | --- |
| 输出格式 | `SVG` 源图与 `PNG` 预览图均已生成 |
| 图片尺寸 | `1600 x 1040` |
| 标书可读性 | 标题、模块层次、注脚在整页预览中清晰可读 |
| 可修改性 | SVG 中的文字、颜色、位置均可继续调整 |
| 内容约束 | 对尚未确认的部署与安全细节通过注脚明确保留确认空间 |

## 结论判定标准

- 图形是否能直接放在标书章节正文中。
- 业务内容是否能够从章节材料准确归纳。
- 是否避免编造未确认的技术架构信息。
- 是否可以沿用同一配色和组件样式批量生成其他插图。

## 本次判断

本图已经达到可放入标书初稿评审的效果。该路线不依赖额外 AI 绘图订阅：大模型根据章节内容生成 SVG，渲染 PNG 用于审阅；确认后保留 SVG 作为高质量插图源文件。后续可沿用同样配色与版式生成业务流程图、部署架构图、数据流转图和安全体系图。
