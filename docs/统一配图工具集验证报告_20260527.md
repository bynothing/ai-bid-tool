## 验证目的

验证同一份招投标文档配图任务能否统一调度原生 SVG 框图和 ECharts HTML 数据图，并生成文档自动插图所需的图件、章节映射与审图页面。

## 混合任务内容

| 图件 ID | 章节位置 | 渲染器 | 图型 | 输出结果 |
| --- | --- | --- | --- | --- |
| `overall_architecture` | `3.1 系统总体架构设计` | `svg` | 分层总体架构图 | SVG / PNG |
| `coverage_trend` | `3.2 建设成效与覆盖能力` | `echarts_html` | 柱线组合图 | SVG / PNG / HTML |
| `data_flow` | `3.3 数据资源汇聚设计` | `echarts_html` | Sankey 流向图 | SVG / PNG / HTML |

## 输出合同验证

本次结果目录生成以下可被后续程序消费的内容：

| 输出 | 用途 |
| --- | --- |
| `illustration-manifest.json` | 文档装配程序按章节位置插图的机器清单 |
| `生成清单.md` | 人工复核图件、题注与插入章节 |
| `assets/svg/` | 专用框图引擎产出的正式图件 |
| `assets/echarts/` | 数据图引擎产出的正式图件 |
| `review/echarts/index.html` | 数据图本地交互审阅与重新导出页面 |

## 结论

现有能力已从两个独立出图原型封装为一个可统一调用的标书配图工具集。后续 AI 只需要生成统一配图任务，文档程序只需要读取结果清单；底层使用 SVG、ECharts 或后续新增 HTML 渲染器，不影响自动插图接口。
