下面把整轮讨论收敛成一个可执行的重构目标和分阶段计划,所有决策点(SVG 模板为主、契约先行生成、复用现有 svg 渲染、绞杀者式扩类)都已经落进去了。

## 重构目标

**北极星:** 把配图工具从"自由生成 SVG"重构为"**冻结 SVG 模板 + 契约先行生成**"的体系,在真实投标内容上稳定产出精美、规范的配图。

**六条不变式(贯穿全程的设计纪律):**

模板 = 冻结的 SVG 几何 + 自描述契约,运行时几何零改动;LLM 只做两件语义活——选模板、按契约生成,其余全部确定性软件;合规在生成时就保证(适配前移),渲染器保持哑;外部契约稳定——job 进、manifest 出,调用方不依赖内部模板/渲染器/文件名;SVG 模板为主力,echarts 只管数据图,graphviz/mermaid 仅兜大型自动布线;主题与几何正交,换色走变量、不动布局。

**本轮非目标(明确不做,避免范围蔓延):** 参考图自动生成模板(后置到规范稳定之后);28 类图型一次性全模板化;替换现有 svg 渲染能力(复用而非重写)。

**量化验收基线:** 选定图型在真实样本上,数据级校验 + 渲染 QA 通过率达标,零文字爆框 / 零元素重叠 / 零穿框;同一张图切换 3 套主题版面不变;每模板回归用例全绿。

## 分阶段计划

### Phase 0 · 地基与止血

目标是主链路干净、抽象就位、能跑测试。先快照当前未提交改动并拉基线分支,把"已有改动"和"重构新增"分开。然后修掉记录里的两个断点:`api.plan()` 返回项带上模板隐含的最终 renderer(`{id, type, decision:{template, renderer, theme, fit_score, fallback?}}`),`api.render()` 增加 `export_echarts` 并透传到 `platform.render_job/render_job_file`。确认 `_fix_fonts.py` 是否在主链路,不在就移出编译门禁或修语法。补上 `pytest` 跑通现有测试作为回归基线。最后只定义抽象、不含具体模板:`Template`(svg + contract)、`Contract`(范式 + 结构 schema + 约束)、`TemplateRegistry` 扩展、manifest 新增字段。

涉及 `api.py`、`platform.py`、`core/registry.py`、`core/manifest.py`、`_fix_fonts.py`。验收:`compileall` 全绿、现有测试通过、新签名有单测、空注册表可 import。

### Phase 1 · 单图型纵向贯通(系统架构图)

目标是把一个图型端到端跑通,在真实内容上产出一张稳定精美的架构图。手工精修 1 个 SVG 模板变体(如 `arch.layered.v2`,3 层),冻结背景/图标/箭头/文字框,槽位用占位标记;为它写契约(范式描述 + 结构 schema:层数、每层节点数、字数上限、图标白名单);登记到注册表(svg + contract + 目录摘要)。把 S5 升级为两段——选模板(LLM 读目录摘要)、按契约生成 job(LLM 读完整 schema)。加确定性校验门禁(必填/字数/节点数/图标白名单),不过就把具体违规项反馈重试。渲染复用 `svg_v2`,把校验过的数据注入冻结模板;主题用 CSS 变量挂一套默认色。S6 读 manifest 插图打通。

涉及 `s5_illustrate.py`、`core/router.py`、`normalize.py`、`renderers/svg_v2.py`、schemas、registry、manifest。验收:用多份真实架构描述生成,零爆框/重叠/穿框;校验失败能自动重试到合规或回退;manifest 字段完整;S6 章节正确插图。**这一阶段是整套方案的可行性验证,务必窄而深。**

### Phase 2 · 规范固化 + 变体阶梯 + 稳定性硬化

把 Phase 1 的经验提炼成《模板与契约规范》(契约必含项、槽位定义法、约束表达、主题变量结构、命名与版本规则)。给架构图补容量变体(3/6 节点、带分组),用确定性 `fit_score` 复核 LLM 的选择。实现固定优先级的回退阶梯:换更大变体 → 拆成两张图 → 通用保底模板 → 标记人工。把渲染后像素 QA(溢出/重叠/对比度)作为最后兜底,接入 `s7_verify`。每个模板配黄金回归快照,纳入 CI。

涉及 `core/router.py`、`core/validator.py`、`s7_verify.py`、测试。验收:规范评审通过;超容量内容能正确走回退;切 3 套主题版面不变;回归全绿。

### Phase 3 · 横向扩类(绞杀者式)

按规范为下一批高频图型(流程图、组织结构、技术路线)各做 1-2 变体 + 契约。`router` 命中 SVG 模板走模板路径,未命中回退现有渲染器,保证老图型零回归。echarts 仅数据图,graphviz/mermaid 仅大型自动布线场景。每类纳入回归。

涉及 registry、router、`renderers/echarts_v2.py`、各 stage。验收:每个新图型达到 Phase 1 同等质量基线;老图型无回归。

### Phase 4 · 工具化与降本(后置 / 可选)

模板制作工具(自动校验契约与 SVG 槽位一致性);"参考图 → 草稿模板"辅助,人机协作产出 `Template` 实例再人工精修——**仅在模板规范稳定后才启动**,绝不前置。

## 排期与风险提示

关键路径是 Phase 1:它一旦跑通,后面都是复制。最大的实现风险也集中在这里——"选模板 + 按契约生成"两段 LLM 交互的提示词工程,以及契约 schema 表达力够不够。建议 Phase 1 只追求"一个变体、一条主链路打通",变体阶梯和回退等到 Phase 2 再补,不要在没验证闭环前就铺开。每个阶段结束都要在真实投标样本上验收,而不是只看示例。
