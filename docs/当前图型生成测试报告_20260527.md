## 4. 各图型效果判断

| 图型 | 适合的标书内容 | 本轮观察 | 结论 |
| --- | --- | --- | --- |
| 分层总体架构图 | 总体设计、平台层级、外部集成 | 信息容量高，层次和图标表达完整 | 可直接作为标书主图类型 |
| 普通流程图 | 单一业务闭环、判断分支 | 节点与箭头清楚；节点少时画布留白偏大 | 可用，后续应增加紧凑布局 |
| 功能能力框图 | 功能响应、模块目录 | 内容检索清楚，视觉偏结构索引 | 适合章节总览，不适合复杂逻辑 |
| 时序图 | 接口调用、告警触发、消息反馈 | 生命线和消息语义清楚，可读性好 | 可直接用于交互说明 |
| 泳道流程图 | 跨部门/跨角色责任流转 | 责任边界直观，颜色语义充分 | 可直接用于业务方案章节 |
| 复杂关系架构图 | 系统域、数据流、告警流、同步关系 | 容器和图例清楚；复杂度上升后需控制连线 | 可用，需设置密度限制 |
| 图标化流程图 | 需要增强识别的小型闭环图 | 图标能提升节点辨识度 | 可作为流程图常规表达 |

## 6. 当前不足与下一阶段测试项

| 项目 | 当前情况 | 后续方向 |
| --- | --- | --- |
| 流程图自动紧凑布局 | 节点少时留白较多 | 服务版引入按节点分布自适应画布 |
| 复杂关系自动避线 | 当前依赖结构化位置及控制边数 | 引入布局/路由模块及遮挡质量检测 |
| 图表与统计图 | 原生 SVG 引擎不实现；本地 ECharts 工作台已补充柱线和雷达图 | 按数据图协议继续扩展 |
| Sankey / 网络图 / 地图 | 本地 ECharts 工作台已补充 Sankey；网络图和地图待扩展 | 进入后续数据图专项测试 |
| 一份标书批量规划图件 | 当前需人工选择样例 | 建立 AI `plan_for_section` 与批量渲染服务 |

## 8. 测试结论

当前原生 SVG 原型已能稳定输出 6 类标书高频框图，并维持一致的净白商务视觉，适合作为专用框图能力的视觉与用例基线。数据型复杂图已由独立的本地 ECharts HTML 工作台补充实现，二者按图件职责分别维护。
