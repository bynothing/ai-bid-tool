# 统一标书配图工具集调用说明

## 1. 工具定位

本工具集服务于本地招投标文档写作过程中的自动配图需求。调用方提交一份与文档章节位置绑定的结构化配图任务，工具根据图件意图选择合适渲染器，输出可插入 Word、PDF 或网页文档的 SVG/PNG，以及后续自动装配所需的清单数据。

当前统一入口：[proposal_illustration_toolkit.py](<../src/proposal_illustration_toolkit.py>)

统一任务 Schema：[proposal_illustration_job.schema.json](<../schema/proposal_illustration_job.schema.json>)

完整示例：[示例_统一文档配图任务.json](<../examples/示例_统一文档配图任务.json>)

## 2. 能力构成

| 能力线 | 统一渲染器标识 | 当前图型 | 交付物 | 使用场景 |
| --- | --- | --- | --- | --- |
| 专用 SVG 框图引擎 | `svg` | 分层架构、流程、泳道、时序、能力、关系图 | SVG / PNG | 方案架构、业务流程、责任关系 |
| ECharts HTML 数据图引擎 | `echarts_html` | 柱线、雷达、Sankey | 离线 HTML + SVG / PNG | 数量趋势、指标评价、数据流向 |

`echarts_html` 的 HTML 页面既是审图工作台，也是图件导出的本地运行载体。后续若需要完整网页式方案图或复合大屏图，可继续在统一任务路由层增加新的 HTML 渲染器，而无需改变文档装配接口。

## 3. 总体调用流程

```text
标书章节文本、数据表、插图位置
          |
          v
AI 配图规划：选择图型并生成统一任务 JSON
          |
          v
proposal_illustration_toolkit.py
  校验章节位置与图件描述 -> 按 renderer 路由
          |
     +----+----+
     |         |
     v         v
 SVG 框图    ECharts HTML 数据图
     |         |
     +----+----+
          v
SVG / PNG / 审图页面 / illustration-manifest.json
          |
          v
文档装配程序依据 section、caption、outputs 自动插图
```

## 4. 统一任务接口

统一任务面向“文档中要放什么图、放在哪里”，而不是面向某个底层渲染库。

```json
{
  "version": "1.0",
  "document": {
    "title": "技术投标方案",
    "projectName": "项目名称",
    "bidSection": "总体技术方案"
  },
  "style": {
    "theme": "clarity_blue",
    "preferredFormat": "both"
  },
  "illustrations": [
    {
      "id": "overall_architecture",
      "renderer": "svg",
      "insertion": {
        "section": "3.1 系统总体架构设计",
        "afterText": "建议插入位置的原文锚点",
        "caption": "图 3-1 系统总体架构图",
        "purpose": "解释平台整体组成"
      },
      "spec": {
        "type": "layered_architecture",
        "title": "系统总体架构图",
        "layers": []
      }
    }
  ]
}
```

关键字段：

| 字段 | 含义 | 自动插图价值 |
| --- | --- | --- |
| `document` | 本次配图所属文档 | 让批量任务归属明确 |
| `style.theme` | SVG 统一配色主题 | 确保同一本标书视觉一致 |
| `style.preferredFormat` | `svg`、`png` 或 `both` | 控制装配产物类型 |
| `illustrations[].id` | 图件稳定标识 | 便于重渲染后替换同一位置图片 |
| `renderer` | `svg` 或 `echarts_html` | 由路由器选择实际生成链路 |
| `insertion.section` | 目标章节 | 文档装配定位依据 |
| `insertion.afterText` | 可选文本锚点 | 用于自动插入到段落之后 |
| `insertion.caption` | 正式图题 | 用于正文题注和清单 |
| `spec` | 对应渲染器的受约束描述 | AI 只表达内容，不控制底层绘制代码 |

## 5. 图型选择规则

| 文档表达目标 | 推荐渲染器 | 推荐图型 |
| --- | --- | --- |
| 描述平台组成、层次与外部接口 | `svg` | `layered_architecture` |
| 描述审批、处置或实施步骤 | `svg` | `flowchart` / `swimlane_flowchart` |
| 描述系统调用或通知时序 | `svg` | `sequence_diagram` |
| 描述多域系统与数据关系 | `svg` | `relationship_map` |
| 描述建设数量与比例趋势 | `echarts_html` | `bar_line` |
| 描述多维能力响应或评分 | `echarts_html` | `radar` |
| 描述数据来源、汇聚与应用方向 | `echarts_html` | `sankey` |

选择原则：有明确节点、连接和职责边界的内容使用 SVG 框图；有统计值、维度比较或流量权重的内容使用 ECharts 数据图。

## 6. 命令调用

在工具目录执行：

```powershell
# 校验 AI 生成的统一任务
python '.\src\proposal_illustration_toolkit.py' `
  --job '.\examples\示例_统一文档配图任务.json' `
  --validate-only

# 生成可用于正式文档的全部图件与清单
python '.\src\proposal_illustration_toolkit.py' `
  --job '.\examples\示例_统一文档配图任务.json'

# Word/PDF 最终定稿输出：提高 PNG 打印精度
python '.\src\proposal_illustration_toolkit.py' `
  --job '.\examples\示例_统一文档配图任务.json' `
  --png-scale 3

# 强制同时输出 PNG，适用于任务中未声明 preferredFormat 的场景
python '.\src\proposal_illustration_toolkit.py' `
  --job '.\examples\示例_统一文档配图任务.json' `
  --png

# 只生成 ECharts 本地审图页，不自动导出其图片
python '.\src\proposal_illustration_toolkit.py' `
  --job '.\examples\示例_统一文档配图任务.json' `
  --no-echarts-export
```

当 `style.preferredFormat` 为 `both` 或 `png` 时，路由器会自动生成 PNG，不需要调用方附加参数。PNG 默认按 `2x` 倍率生成；Word 等不能直接使用 SVG 的装配链路，应保留高分辨率 PNG，并在定稿导出时关闭 Office 图片压缩。

## 7. 输出目录合同

默认输出目录为 `results/current/统一配图任务/`：

```text
统一配图任务/
|-- 生成清单.md                       # 供人工审阅的图件与插图位置清单
|-- illustration-manifest.json       # 供文档自动装配程序读取
|-- inputs/
|   |-- svg-diagrams.json            # 路由后交给 SVG 引擎的输入
|   `-- echarts-diagrams.json        # 路由后交给 ECharts 页面的输入
|-- assets/
|   |-- svg/                         # 正式框图 SVG/PNG
|   `-- echarts/                     # 正式数据图 SVG/PNG
`-- review/
    `-- echarts/
        `-- index.html               # 数据图离线审图页面
```

`illustration-manifest.json` 中每张图均保留：

- `id`：稳定替换标识。
- `renderer`：实际出图链路。
- `section` 和 `caption`：插入位置与正式题注。
- `outputs.svg`、`outputs.png`：文档装配应引用的文件。
- `outputs.reviewHtml`：需要复核数据图时可打开的页面。

## 8. AI 统一生成约束

面向大模型的调用应坚持以下约束：

1. 先判断段落是否真正需要插图，再决定图型。
2. 所有图件必须指定章节和正式题注，不能只生成孤立图片。
3. SVG 框图仅生成结构、节点和关系描述，不生成 SVG path 或坐标。
4. ECharts 数据图仅生成结构化数据描述，不生成 JavaScript 或任意 `option`。
5. 标书中没有给出真实数值时，数据图必须标注为演示或待确认数据。
6. 自动插入文档前，应核查图题、来源说明、数值口径和图片可读性。

## 9. 扩展接口

统一路由层用于隔离后续新增能力，推荐按以下方式扩展：

| 扩展需求 | 实现位置 | 对调用方影响 |
| --- | --- | --- |
| SVG 新图型或图标 | `svg_diagram.schema.json` 与 SVG 引擎 | `renderer: svg` 不变 |
| ECharts 新统计图 | `echarts_diagram.schema.json` 与工作台构建器 | `renderer: echarts_html` 不变 |
| 完整 HTML 复合页面 | 新增 HTML renderer 与独立 Schema | 增加新的 `renderer` 标识 |
| Word/Markdown 自动插图 | 读取 `illustration-manifest.json` 的装配模块 | 不改变现有渲染器 |

因此，后续 AI 或文档程序只需要稳定调用统一任务协议和结果清单，不需要掌握各个底层绘图实现。
